# AI 改卷 / 试卷讲解 Web 项目需求文档（初稿）

## 1. 文档目的

本文档用于定义“AI 改卷 / 试卷讲解 Web”项目的第一版需求范围，作为后续产品设计、技术设计、任务拆解与开发实现的基础。

当前目标是先完成一个可运行的 MVP 版本，形成从“上传题目图片”到“AI 引导讲解 + 画布标注 + 会话保存/回放”的最小业务闭环。

---

## 2. 项目背景

传统拍题产品大多直接给出答案，不利于学生形成解题思路。本项目希望构建一个更像“老师带着学生一步步思考”的 Web 应用，让 AI 在识别题目后，以引导式方式帮助学生理解题意、拆解步骤、定位错误、逐步接近答案，而不是一次性给出最终结果。

同时，由于题目具有图片、几何图形、批注、演示过程等特点，系统需要提供画布能力，支持 AI 和用户都围绕题目图片进行标注和辅助讲解。

---

## 3. 项目目标

### 3.1 总体目标

构建一个支持题目图片上传、多模型选择、引导式 AI 讲解、画布标注、会话记录和对话重放的 Web 应用。

### 3.2 MVP 目标

第一版聚焦最小闭环：

- 用户可以注册登录
- 用户可以上传单张题目图片
- 用户可以选择不同大模型发起提问
- AI 根据题图进行引导式讲解
- AI 可以返回基础标注信息，在题图上高亮、框选、箭头提示、文字批注
- 用户可以在题图画布上继续手工标注
- 系统可以保存会话历史并支持重放

### 3.3 非目标

以下内容不作为第一版必须目标：

- 自动批量改整张试卷
- 复杂教师后台与运营后台
- 多角色复杂权限体系
- 大规模班级管理
- 精细化计费系统
- 复杂知识图谱推荐

---

## 4. 用户角色

### 4.1 学生用户

核心使用者，上传题目图片，与 AI 多轮交互，在画布中查看和补充标注，回看历史讲解过程。

### 4.2 管理员

第一版可弱化，仅负责模型配置、基础内容管理、查看运行状态。

说明：

- 第一版实现中可先只完整支持学生端
- 管理端可以先提供接口，页面可后补

---

## 5. 使用场景

### 5.1 题目讲解

学生上传数学题、几何题、英语题等图片，选择一个支持视觉理解的大模型，请 AI 帮助讲解。AI 不直接给出答案，而是先帮助学生理解题目、定位关键信息、提示下一步思路。

### 5.2 错题订正

学生上传自己做过并已标注痕迹的题目，AI 分析错误位置，在图中圈出关键区域，并给出引导性问题，例如“这一步公式为什么可以这样变形？”。

### 5.3 几何图辅助说明

AI 在原题图上绘制辅助线、箭头、框选区域或文字标注，帮助学生理解几何关系。

### 5.4 历史复盘

学生回看过去一次完整对话，包括题图、聊天内容、AI 标注步骤和自己的画布操作，复习解题过程。

---

## 6. 产品原则

### 6.1 引导优先

AI 的默认角色是“引导型老师”，优先通过提问、提示、分步启发帮助学生思考，而不是立刻输出完整答案。

### 6.2 图文结合

题目理解、讲解和解释不能只依赖文字，需要结合图片区域定位、标注、框选和绘图。

### 6.3 可追溯

系统应保存一次讲解的关键过程，使用户能够回看 AI 如何一步步引导。

### 6.4 模型可切换

系统应支持不同大模型的接入与切换，便于比较效果、控制成本和后续扩展。

---

## 7. 功能范围

### 7.1 账户与登录

功能描述：

- 支持用户注册
- 支持账号密码登录
- 支持登录状态保持
- 支持退出登录
- 支持查看当前用户信息

MVP 要求：

- 先使用账号 + 密码方式
- 密码需加密存储
- 使用 Token 或 Session 保持登录态

验收标准：

- 未登录用户不能访问会话和上传接口
- 登录成功后可以访问个人数据

### 7.2 题目图片上传

功能描述：

- 用户上传题目图片
- 系统校验图片格式与大小
- 系统保存原图和元数据
- 上传后可立即用于创建 AI 会话

MVP 要求：

- 支持单张图片上传
- 支持常见图片格式：`jpg`、`jpeg`、`png`、`webp`
- 对超大图片进行限制或压缩

验收标准：

- 上传成功后能返回图片 ID 和访问地址
- 非法文件不能上传成功

### 7.3 模型选择与 AI 对话

功能描述：

- 用户在发起对话时可以选择不同大模型
- 后端根据模型配置将请求路由到不同供应商
- 支持题图 + 文本问题联合输入
- 支持多轮对话

MVP 要求：

- 至少接入 2 个支持视觉输入的大模型
- 提供模型列表接口
- 会话中记录本次所用模型

验收标准：

- 用户可在前端明确选择模型
- 相同题图可用不同模型发起会话

### 7.4 引导型老师回答策略

功能描述：

- AI 以老师身份进行启发式讲解
- 不默认直接输出完整答案
- 在多轮交互中根据学生反馈逐步推进

行为要求：

- 优先解释题意、已知条件、求解目标
- 优先提出引导问题
- 优先给出下一步提示而非整题完整过程
- 当学生明显卡住时，可以逐步增加提示强度
- 在合适条件下可以给出总结，但不应一开始直接抄最终答案

示例风格：

- “先不要急着算结果，我们先看题目中已知了什么。”
- “你觉得这一步最适合用哪个公式？”
- “先圈出题目中的关键条件，我们一起看。”

验收标准：

- AI 首轮回复以引导为主，而非直接给终答案
- 多轮对话中能根据上下文继续启发

### 7.5 画布功能

功能描述：

- 在题目图片上提供可交互画布
- 用户可以手工进行标注和绘图
- AI 返回的标注可以渲染到画布上

MVP 工具范围：

- 画笔
- 矩形框
- 箭头
- 文本标注
- 橡皮擦
- 撤销
- 重做
- 缩放
- 平移

图层要求：

- 原题图片层
- AI 标注层
- 用户标注层

验收标准：

- 用户可以在题图上完成基本标注
- AI 标注与用户标注可以同时显示

### 7.6 AI 标注与画图

功能描述：

- AI 在回答时除文本外，还可返回结构化标注信息
- 前端根据结构化数据，在画布上进行高亮、框选、箭头或文字说明

MVP 标注类型：

- 高亮某区域
- 矩形框选
- 箭头指向
- 简短文字批注

扩展预留：

- 几何辅助线
- 圆形标注
- 分步骤动画标注

验收标准：

- AI 标注能随消息一起展示
- 无效标注数据不会导致页面崩溃

### 7.7 会话历史

功能描述：

- 系统保存用户与 AI 的会话
- 用户可以查看会话列表
- 用户可以进入某个历史会话继续对话

MVP 要求：

- 保存标题、模型、关联图片、消息内容、标注数据
- 支持删除会话

验收标准：

- 刷新页面后仍能看到历史记录
- 重新打开会话后能继续对话

### 7.8 对话重放

功能描述：

- 回放一次题目讲解的过程
- 展示消息顺序和画布标注变化

MVP 要求：

- 至少支持消息逐条展示
- 支持 AI 标注按步骤重放
- 用户画布操作可先保存快照或操作日志，二选一

验收标准：

- 用户能清晰回看一次讲解过程
- 回放结果与原会话核心内容一致

## 8. 业务流程

### 8.1 首次使用流程

1. 用户注册或登录
2. 进入主页面
3. 上传题目图片
4. 选择模型
5. 输入问题或直接请求讲解
6. AI 返回引导式文本与标注
7. 用户在画布上查看并补充标注
8. 用户继续追问
9. 系统保存会话

### 8.2 历史复盘流程

1. 用户进入历史会话列表
2. 选择某次题目讲解
3. 查看完整对话
4. 点击回放
5. 系统逐步展示消息和标注过程

## 11. AI 输出协议要求

### 11.1 输出内容组成

AI 返回内容建议拆成两部分：

- 面向用户的自然语言讲解
- 面向前端的结构化标注指令

### 11.2 建议结构

建议由后端统一解析以下结构：

```json
{
  "replyText": "我们先看已知条件，题目里给了两个角度信息。",
  "guidanceStage": "observe",
  "annotations": [
    {
      "type": "rect",
      "x": 120,
      "y": 80,
      "width": 200,
      "height": 90,
      "color": "#ff6b6b",
      "label": "先看这个条件"
    }
  ]
}
```

### 11.3 协议原则

- 即使模型输出不完全符合 JSON，也要有兜底解析策略
- 前端只渲染受支持的标注类型
- 不可信字段不得直接执行

## 12. 非功能需求

### 12.1 安全性

- 密码必须加密存储
- 上传文件需做格式与大小校验
- 关键接口需鉴权
- 用户只能访问自己的会话和资源
- 模型 API Key 必须通过配置管理，不可硬编码

### 12.2 性能

- 普通图片上传应在可接受时间内完成
- 单次 AI 请求需具备超时控制
- 画布操作要保持基本流畅

第一版建议目标：

- 普通图片上传时间小于 3 秒
- 非流式 AI 首次响应目标小于 10 秒

### 12.3 可扩展性

- 模型供应商应可插拔
- 画布标注类型应可扩展
- 未来可支持 OCR、整卷拆题、教师端等能力

### 12.4 可观测性

- 记录模型调用耗时、成功率、失败原因
- 记录上传失败、解析失败、标注失败等关键事件

- 用户可以完成注册、登录、退出

## 15. 风险与难点

### 15.1 模型输出不稳定

不同模型对结构化输出的遵循程度不同，可能影响标注解析与前端渲染。

应对建议：

- 使用强约束 Prompt
- 后端增加容错解析
- 标注协议保持简单

### 15.2 图片坐标映射复杂

原图尺寸、前端缩放比例、移动端适配都可能导致标注坐标偏移。

应对建议：

- 前后端统一基于原图坐标系存储
- 前端仅负责缩放映射显示

### 15.3 画布回放实现复杂

如果直接保存每一步操作，回放实现较复杂；如果只存快照，精细回放能力有限。

应对建议：

- MVP 可优先存关键快照 + AI 标注步骤
- 二期再增强为完整操作日志回放
